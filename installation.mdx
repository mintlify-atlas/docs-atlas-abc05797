# Installation

This guide covers how to add ZIO to your Scala project using different build tools and Scala versions.

## Requirements

ZIO requires:
- **Java 8** or higher
- **Scala 2.12.21**, **2.13.18**, or **3.3.7**

ZIO has **zero external dependencies**, making it lightweight and easy to integrate.

## Quick Start

<Tabs>
  <Tab title="sbt">
    Add this line to your `build.sbt`:

    ```scala build.sbt
    libraryDependencies += "dev.zio" %% "zio" % "2.1.0"
    ```
  </Tab>
  
  <Tab title="Mill">
    Add this to your `build.sc`:

    ```scala build.sc
    def ivyDeps = Agg(
      ivy"dev.zio::zio:2.1.0"
    )
    ```
  </Tab>
  
  <Tab title="Gradle">
    Add this to your `build.gradle`:

    ```groovy build.gradle
    dependencies {
      implementation 'dev.zio:zio_3:2.1.0'
    }
    ```
    
    <Note>Replace `_3` with `_2.13` or `_2.12` for Scala 2.x versions.</Note>
  </Tab>
</Tabs>

## Installation by Build Tool

### sbt (Scala Build Tool)

sbt is the most popular build tool for Scala projects.

<Steps>

### Create or open your `build.sbt` file

In your project root, create or edit `build.sbt`.

### Add ZIO dependency

```scala build.sbt
val zioVersion = "2.1.0"

scalaVersion := "3.3.7"  // or "2.13.18" or "2.12.21"

libraryDependencies += "dev.zio" %% "zio" % zioVersion
```

### Add additional ZIO libraries (optional)

```scala build.sbt
libraryDependencies ++= Seq(
  "dev.zio" %% "zio"         % zioVersion,
  "dev.zio" %% "zio-streams" % zioVersion,
  "dev.zio" %% "zio-test"    % zioVersion % Test,
  "dev.zio" %% "zio-test-sbt" % zioVersion % Test
)
```

### Enable test framework (for ZIO Test)

If using ZIO Test:

```scala build.sbt
testFrameworks += new TestFramework("zio.test.sbt.ZTestFramework")
```

</Steps>

<Tip>
  The `%%` operator automatically appends the Scala version to the artifact name. Use `%%%` for cross-platform projects (JVM, JS, Native).
</Tip>

### Complete sbt Example

Here's a complete `build.sbt` for a ZIO project:

```scala build.sbt
name := "my-zio-app"

version := "0.1.0"

scalaVersion := "3.3.7"

val zioVersion = "2.1.0"

libraryDependencies ++= Seq(
  "dev.zio" %% "zio"          % zioVersion,
  "dev.zio" %% "zio-streams"  % zioVersion,
  "dev.zio" %% "zio-test"     % zioVersion % Test,
  "dev.zio" %% "zio-test-sbt" % zioVersion % Test
)

testFrameworks += new TestFramework("zio.test.sbt.ZTestFramework")

// Optional: Enable better stack traces
scalacOptions += "-Ypatmat-exhaust-depth"
scalacOptions += "40"
```

## Installation by Scala Version

### Scala 3.3.x

ZIO fully supports Scala 3 with all modern features:

```scala build.sbt
scalaVersion := "3.3.7"

libraryDependencies += "dev.zio" %% "zio" % "2.1.0"
```

### Scala 2.13.x

The recommended Scala 2 version:

```scala build.sbt
scalaVersion := "2.13.18"

libraryDependencies += "dev.zio" %% "zio" % "2.1.0"
```

### Scala 2.12.x

For projects that must use Scala 2.12:

```scala build.sbt
scalaVersion := "2.12.21"

libraryDependencies += "dev.zio" %% "zio" % "2.1.0"
```

<Note>
  While ZIO supports Scala 2.12, we recommend using Scala 2.13 or Scala 3 for new projects.
</Note>

## Cross-Platform Projects

ZIO supports cross-platform development across JVM, JavaScript, and Native.

### Scala.js

To use ZIO with Scala.js:

```scala build.sbt
enablPlugins(ScalaJSPlugin)

scalaVersion := "3.3.7"

libraryDependencies += "dev.zio" %%% "zio" % "2.1.0"
```

<Tip>
  Note the `%%%` operator which automatically selects the correct platform artifact.
</Tip>

### Scala Native

To use ZIO with Scala Native:

```scala build.sbt
enablePlugins(ScalaNativePlugin)

scalaVersion := "3.3.7"

libraryDependencies += "dev.zio" %%% "zio" % "2.1.0"
```

### Cross-Build Configuration

For a project that compiles to all three platforms:

```scala build.sbt
import sbtcrossproject.CrossPlugin.autoImport.{crossProject, CrossType}

lazy val myProject = crossProject(JSPlatform, JVMPlatform, NativePlatform)
  .in(file("."))
  .settings(
    scalaVersion := "3.3.7",
    libraryDependencies += "dev.zio" %%% "zio" % "2.1.0"
  )
```

## Mill Build Tool

Mill is a modern build tool for Scala:

```scala build.sc
import mill._, scalalib._

object myproject extends ScalaModule {
  def scalaVersion = "3.3.7"
  
  def ivyDeps = Agg(
    ivy"dev.zio::zio:2.1.0",
    ivy"dev.zio::zio-streams:2.1.0"
  )
  
  object test extends Tests {
    def ivyDeps = Agg(
      ivy"dev.zio::zio-test:2.1.0",
      ivy"dev.zio::zio-test-sbt:2.1.0"
    )
    
    def testFramework = "zio.test.sbt.ZTestFramework"
  }
}
```

## Gradle Build Tool

For Gradle projects (using the Scala plugin):

```groovy build.gradle
plugins {
    id 'scala'
    id 'application'
}

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.scala-lang:scala3-library_3:3.3.7'
    implementation 'dev.zio:zio_3:2.1.0'
    implementation 'dev.zio:zio-streams_3:2.1.0'
    
    testImplementation 'dev.zio:zio-test_3:2.1.0'
    testImplementation 'dev.zio:zio-test-sbt_3:2.1.0'
}

application {
    mainClass = 'com.example.Main'
}
```

<Note>
  Remember to replace `_3` with `_2.13` or `_2.12` based on your Scala version.
</Note>

## Snapshots

To use snapshot versions for bleeding-edge features:

```scala build.sbt
resolvers += "Sonatype OSS Snapshots" at 
  "https://oss.sonatype.org/content/repositories/snapshots"

libraryDependencies += "dev.zio" %% "zio" % "2.1.0+XX-SNAPSHOT"
```

<Warning>
  Snapshot versions are unstable and should not be used in production.
</Warning>

## Verify Installation

Create a simple test file to verify ZIO is installed correctly:

```scala src/main/scala/VerifyInstall.scala
import zio._

object VerifyInstall extends ZIOAppDefault {
  def run = 
    for {
      _ <- Console.printLine("ZIO is installed correctly!")
      _ <- Console.printLine(s"ZIO Version: ${zio.BuildInfo.version}")
    } yield ()
}
```

Run it:

```bash
sbt run
```

You should see:

```
ZIO is installed correctly!
ZIO Version: 2.1.0
```

## Troubleshooting

### Dependency Resolution Issues

If you encounter dependency conflicts:

```scala build.sbt
// Force a specific version
dependencyOverrides += "dev.zio" %% "zio" % "2.1.0"
```

### Scala Version Mismatches

Ensure all ZIO libraries use the same version:

```scala build.sbt
val zioVersion = "2.1.0"

libraryDependencies ++= Seq(
  "dev.zio" %% "zio"         % zioVersion,
  "dev.zio" %% "zio-streams" % zioVersion,
  "dev.zio" %% "zio-test"    % zioVersion % Test
)
```

### OutOfMemoryError

Increase JVM heap size:

```bash
export SBT_OPTS="-Xmx2G -Xss2M"
sbt run
```

Or in `.sbtopts` file:

```
-Xmx2G
-Xss2M
```

## IDE Setup

### IntelliJ IDEA

1. Install the **Scala plugin**
2. Import your project as an sbt project
3. Wait for indexing to complete
4. Right-click `build.sbt` → **Reload sbt project**

### Visual Studio Code

1. Install the **Metals** extension
2. Open your project folder
3. When prompted, import the build
4. Wait for Metals to finish setting up

### Vim/Neovim

1. Install **metals-vim** or **coc-metals**
2. Open a Scala file
3. Run `:MetalsInstall` to set up the language server
