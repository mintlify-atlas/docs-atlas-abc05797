# Quickstart Guide

This guide will walk you through creating your first ZIO application from scratch. You'll learn how to set up a project, write a simple ZIO program, and run it.

## Prerequisites

Before you begin, make sure you have:

- **Java JDK 8 or higher** installed
- **sbt** (Scala Build Tool) installed - [Download sbt](https://www.scala-sbt.org/download.html)
- A text editor or IDE (we recommend IntelliJ IDEA with the Scala plugin)

## Create Your First ZIO Application

Let's build a simple console application that greets users by name.

<Steps>

### Create a new sbt project

Create a new directory and initialize an sbt project:

```bash
mkdir hello-zio
cd hello-zio
```

Create a file `build.sbt` with the following content:

```scala build.sbt
val zioVersion = "2.1.0"

scalaVersion := "3.3.7"

libraryDependencies += "dev.zio" %% "zio" % zioVersion
```

<Tip>
  You can also use Scala 2.13 or 2.12. Just change the `scalaVersion` to `"2.13.18"` or `"2.12.21"`.
</Tip>

### Create the source directory structure

Create the necessary directories:

```bash
mkdir -p src/main/scala
```

### Write your first ZIO program

Create a file `src/main/scala/HelloZIO.scala` with this code:

```scala src/main/scala/HelloZIO.scala
import zio._

object HelloZIO extends ZIOAppDefault {
  
  def run = 
    for {
      _    <- Console.printLine("Hello! What is your name?")
      name <- Console.readLine
      _    <- Console.printLine(s"Hello, $name! Welcome to ZIO.")
    } yield ()
}
```

Let's break down what's happening here:

- **`extends ZIOAppDefault`** - This makes your object a runnable ZIO application
- **`def run`** - The entry point of your ZIO application
- **`for { ... } yield ()`** - A for-comprehension that sequences ZIO effects
- **`Console.printLine`** - Prints a line to the console (returns `ZIO[Any, IOException, Unit]`)
- **`Console.readLine`** - Reads a line from the console (returns `ZIO[Any, IOException, String]`)

### Run your application

Run the application using sbt:

```bash
sbt run
```

You should see:

```
Hello! What is your name?
```

Type your name and press Enter. You'll see:

```
Hello, [Your Name]! Welcome to ZIO.
```

</Steps>

<Note>
  Congratulations! You've just written and run your first ZIO application.
</Note>

## Understanding the Code

Let's understand the key concepts in our Hello World program:

### The ZIO Effect Type

Every operation in our program returns a `ZIO` effect:

```scala
ZIO[R, E, A]
```

Where:
- **`R`** = Environment type (what the effect needs to run)
- **`E`** = Error type (what errors can occur)  
- **`A`** = Success type (what value is produced)

For example:
```scala
Console.printLine("Hello"): ZIO[Any, IOException, Unit]
```

This effect:
- Requires no environment (`Any`)
- Can fail with `IOException`
- Produces `Unit` (nothing) on success

### Composing Effects with For-Comprehensions

The `for { ... } yield` syntax allows you to compose multiple effects sequentially:

```scala
for {
  a <- effect1  // Run effect1, bind result to 'a'
  b <- effect2  // Run effect2, bind result to 'b'  
  c <- effect3  // Run effect3, bind result to 'c'
} yield result  // Return final result
```

Each step runs after the previous one completes successfully.

## A More Advanced Example

Let's build a simple number guessing game:

```scala src/main/scala/GuessingGame.scala
import zio._
import scala.util.Random

object GuessingGame extends ZIOAppDefault {
  
  def analyzeGuess(secret: Int, guess: Int): UIO[String] =
    if (guess < secret) ZIO.succeed("Too low!")
    else if (guess > secret) ZIO.succeed("Too high!")
    else ZIO.succeed("Correct!")
  
  def gameLoop(secret: Int, attempts: Int): ZIO[Any, IOException, Unit] =
    for {
      _     <- Console.printLine(s"Attempt #$attempts - Guess a number between 1 and 100:")
      input <- Console.readLine
      guess <- ZIO.attempt(input.toInt).catchAll(_ => 
                 Console.printLine("Please enter a valid number") *> gameLoop(secret, attempts)
               )
      hint  <- analyzeGuess(secret, guess)
      _     <- Console.printLine(hint)
      _     <- if (guess == secret) 
                 Console.printLine(s"You won in $attempts attempts!")
               else 
                 gameLoop(secret, attempts + 1)
    } yield ()
  
  def run = 
    for {
      _      <- Console.printLine("Welcome to the Number Guessing Game!")
      secret <- ZIO.succeed(Random.nextInt(100) + 1)
      _      <- gameLoop(secret, 1)
    } yield ()
}
```

This example demonstrates:

- **Error handling** with `catchAll`
- **Recursion** for game loops
- **Conditional logic** within ZIO effects
- **Pure values** with `ZIO.succeed`
- **Effect sequencing** with `*>` operator

## Working with Errors

ZIO makes error handling explicit and type-safe:

```scala
import zio._
import java.io.IOException

object ErrorHandlingExample extends ZIOAppDefault {
  
  // An effect that might fail
  val riskyOperation: ZIO[Any, IOException, String] =
    ZIO.attempt(scala.io.Source.fromFile("data.txt").mkString)
      .refineToOrDie[IOException]
  
  // Handle the error
  val safeOperation: ZIO[Any, Nothing, String] =
    riskyOperation.catchAll { error =>
      Console.printLine(s"Error reading file: ${error.getMessage}").orDie *>
      ZIO.succeed("default data")
    }
  
  def run = safeOperation.flatMap(data => Console.printLine(s"Data: $data"))
}
```

## Concurrent Operations

ZIO makes it easy to run effects in parallel:

```scala
import zio._

object ConcurrentExample extends ZIOAppDefault {
  
  def fetchUser(id: Int): UIO[String] = 
    ZIO.succeed(s"User-$id").delay(1.second)
  
  def fetchPosts(userId: String): UIO[List[String]] =
    ZIO.succeed(List(s"Post 1 by $userId", s"Post 2 by $userId")).delay(1.second)
  
  def run = 
    for {
      // Run three user fetches in parallel
      users <- ZIO.collectAllPar(List(
                 fetchUser(1),
                 fetchUser(2),
                 fetchUser(3)
               ))
      _     <- Console.printLine(s"Fetched users: ${users.mkString(", ")}")
      
      // Fetch posts for all users in parallel  
      posts <- ZIO.foreachPar(users)(fetchPosts)
      _     <- Console.printLine(s"Fetched ${posts.flatten.size} posts")
    } yield ()
}
```

This example runs in about 2 seconds instead of 6, thanks to parallel execution.

## Common Patterns

Here are some common operations you'll use frequently:

### Creating Effects

```scala
// Pure value
ZIO.succeed(42)

// Lazy evaluation
ZIO.attempt(println("Hello"))

// From Future
ZIO.fromFuture { implicit ec => Future.successful(42) }

// Async callback
ZIO.async[Any, Throwable, String] { callback =>
  asyncOperation { result =>
    callback(ZIO.succeed(result))
  }
}
```

### Transforming Effects

```scala
val effect: ZIO[Any, Nothing, Int] = ZIO.succeed(21)

// Map the success value
effect.map(_ * 2)  // ZIO[Any, Nothing, Int] = 42

// FlatMap to chain effects
effect.flatMap(n => ZIO.succeed(n * 2))

// Map errors
effect.mapError(_.getMessage)

// Transform both success and failure
effect.fold(
  error => "Error occurred",
  value => s"Success: $value"
)
```

### Running Effects

```scala
import zio._

// In ZIOAppDefault, just define 'run'
object MyApp extends ZIOAppDefault {
  def run = Console.printLine("Hello!")
}

// Or create a custom runtime
val runtime = Runtime.default
val result = Unsafe.unsafe { implicit unsafe =>
  runtime.unsafe.run(ZIO.succeed(42))
}
```

<Warning>
  Only call `unsafe.run` at the edges of your application. Keep your business logic purely functional.
</Warning>
