---
title: ZChannel
---

A `ZChannel[Env, InErr, InElem, InDone, OutErr, OutElem, OutDone]` is a nexus of I/O operations that supports both reading and writing. Channels are the foundation of ZIO Streams: both streams and sinks are built on channels.

Most users shouldn't need to use channels directly, as streams and sinks are more convenient and cover all common use cases. However, when adding new stream and sink operators, or doing something highly specialized, channels provide the low-level primitives needed.

## Type Signature

```scala
sealed trait ZChannel[-Env, -InErr, -InElem, -InDone, +OutErr, +OutElem, +OutDone]
```

The type parameters represent:
- `Env` - Environment required by the channel
- `InErr` - Type of upstream errors the channel can read
- `InElem` - Type of elements the channel can read
- `InDone` - Type of upstream done signal
- `OutErr` - Type of errors the channel may produce
- `OutElem` - Type of elements the channel may produce
- `OutDone` - Type of the terminal value when the channel completes

## Core Concepts

A channel:
- May **read** values of type `InElem` or process upstream failures of type `InErr`
- May **write** values of type `OutElem`
- **Yields** a value of type `OutDone` when finished
- May **fail** with a value of type `OutErr`

## Notes

Channels provide the low-level building blocks for streams and sinks. They offer fine-grained control over:
- Reading and writing operations
- Error propagation
- Resource management
- Concurrent composition

For most use cases, prefer using `ZStream`, `ZSink`, and `ZPipeline` which provide a higher-level API built on channels.
