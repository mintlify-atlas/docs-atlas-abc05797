---
title: ZPipeline
---

A `ZPipeline[Env, Err, In, Out]` is a polymorphic stream transformer. Pipelines accept a stream as input and return a transformed stream as output.

Pipelines can be thought of as a recipe for calling a bunch of methods on a source stream to yield a new (transformed) stream. They separate the stream transformation from the source stream itself, making it possible to abstract over stream transformations at the level of values.

## Type Signature

```scala
final class ZPipeline[-Env, +Err, -In, +Out] private (
  val channel: ZChannel[Env, ZNothing, Chunk[In], Any, Err, Chunk[Out], Any]
)
```

The type parameters are:
- `Env` - Environment required by the pipeline
- `Err` - Error type the pipeline may produce
- `In` - Type of input elements
- `Out` - Type of output elements

## Mental Model

A pipeline can be thought of as having this type (though this exact encoding is not used for inference reasons):

```scala
type ZPipeline[Env, Err, In, Out] = ZStream[Env, Err, In] => ZStream[Env, Err, Out]
```

## Core Constructors

### Identity & Basic

#### `identity`
```scala
def identity[In](implicit trace: Trace): ZPipeline[Any, Nothing, In, In]
```
The identity pipeline that passes through all elements unchanged.

#### `apply`
```scala
def apply[In](implicit trace: Trace): ZPipeline[Any, Nothing, In, In]
```
A shorter version of `identity`, which facilitates more compact pipeline definitions.

```scala
ZPipeline[Int] >>> ZPipeline.filter(_ % 2 != 0)
```

### Construction

#### `fromFunction`
```scala
def fromFunction[Env, Err, In, Out](
  f: ZStream[Env, Err, In] => ZStream[Env, Err, Out]
): ZPipeline[Env, Err, In, Out]
```
Creates a pipeline from a function that transforms streams.

#### `fromSink`
```scala
def fromSink[Env, Err, In, L, Z](
  sink: => ZSink[Env, Err, In, L, Z]
)(implicit trace: Trace): ZPipeline[Env, Err, In, Z]
```
Creates a pipeline from a sink. The pipeline repeatedly runs the sink and emits its results.

#### `fromChannel`
```scala
def fromChannel[Env, Err, In, Out](
  channel: => ZChannel[Env, ZNothing, Chunk[In], Any, Err, Chunk[Out], Any]
)(implicit trace: Trace): ZPipeline[Env, Err, In, Out]
```
Creates a pipeline from a channel.

### Aggregation

#### `aggregateAsync`
```scala
def aggregateAsync[Env, Err, In, Out](
  sink: => ZSink[Env, Err, In, In, Out]
)(implicit trace: Trace): ZPipeline[Env, Err, In, Out]
```
Aggregates elements using the provided sink for as long as downstream operators are busy. Creates two asynchronous processing islands.

#### `aggregateAsyncWithin`
```scala
def aggregateAsyncWithin[Env, Err, In, Out](
  sink: => ZSink[Env, Err, In, In, Out],
  schedule: => Schedule[Env, Option[Out], Any]
)(implicit trace: Trace): ZPipeline[Env, Err, In, Out]
```
Aggregates elements using a sink until it completes or until the delay signaled by the schedule has passed.

#### `aggregateAsyncWithinEither`
```scala
def aggregateAsyncWithinEither[Env, Err, In, Out, Out2](
  sink: => ZSink[Env, Err, In, In, Out],
  schedule: => Schedule[Env, Option[Out], Out2]
)(implicit trace: Trace): ZPipeline[Env, Err, In, Either[Out2, Out]]
```
Like `aggregateAsyncWithin` but returns both schedule outputs (Left) and sink outputs (Right).

#### `takeWhile`

#### `drop`

#### `dropWhile`

## Pipeline Composition

### Applying a Pipeline

```scala
def apply[Env1 <: Env, Err1 >: Err](
  stream: => ZStream[Env1, Err1, In]
)(implicit trace: Trace): ZStream[Env1, Err1, Out]
```
Attaches this pipeline to the given stream.

### Composing Pipelines

#### `>>>`
```scala
def >>>[Env1 <: Env, Err1 >: Err, Out2](
  that: => ZPipeline[Env1, Err1, Out, Out2]
)(implicit trace: Trace): ZPipeline[Env1, Err1, In, Out2]
```
Composes two pipelines by first applying this pipeline, then applying the specified pipeline.

#### `<<<`
```scala
def <<<[Env1 <: Env, Err1 >: Err, In2](
  that: => ZPipeline[Env1, Err1, In2, In]
)(implicit trace: Trace): ZPipeline[Env1, Err1, In2, Out]
```
Composes two pipelines by first applying the specified pipeline, then applying this pipeline.

#### `andThen`
```scala
def andThen[Env1 <: Env, Err1 >: Err, Out2](
  that: => ZPipeline[Env1, Err1, Out, Out2]
)(implicit trace: Trace): ZPipeline[Env1, Err1, In, Out2]
```
A named version of the `>>>` operator.

### Composing with Sinks

```scala
def >>>[Env1 <: Env, Err1 >: Err, Leftover, Out2](
  that: => ZSink[Env1, Err1, Out, Leftover, Out2]
)(implicit trace: Trace): ZSink[Env1, Err1, In, Leftover, Out2]
```
Composes this pipeline with a sink, resulting in a sink that processes elements through the pipeline first.

## Common Patterns

### Text Processing
```scala
val linesPipeline = 
  ZPipeline.utf8Decode >>> 
  ZPipeline.splitLines

stream >>> linesPipeline
```

### Batching with Timeout
```scala
val batchPipeline = 
  ZPipeline.groupedWithin(100, 1.second)

stream >>> batchPipeline
```

### Deduplication
```scala
val dedupPipeline = 
  ZPipeline.changes

stream >>> dedupPipeline
```

### Compression
```scala
val compressPipeline = 
  ZPipeline.utf8Encode >>> 
  ZPipeline.gzip()

stream >>> compressPipeline
```
