title: FiberRef
description: Thread-local storage for fibers - FiberRef provides fiber-local state that's automatically propagated to child fibers and merged on join.

A **FiberRef** is ZIO's equivalent of Java's `ThreadLocal`. The value of a `FiberRef` is automatically propagated to child fibers when they are forked and merged back into the parent fiber after they are joined.

## Basic Concept

Unlike a `Ref`, which provides shared mutable state, a `FiberRef` provides fiber-local state. Each fiber has its own isolated value that doesn't interfere with other fibers.

```scala
import zio._

val program = for {
  fiberRef <- FiberRef.make("Hello world!")
  child    <- fiberRef.set("Hi!").fork
  _        <- child.join
  result   <- fiberRef.get
} yield result
// result = "Hi!" (child's value merged back)
```

## Creating FiberRefs

### Basic Creation

```scala
import zio._

// Create with initial value
val fiberRef: ZIO[Scope, Nothing, FiberRef[String]] = 
  FiberRef.make("initial value")

// Create with custom fork and join logic
val customRef: ZIO[Scope, Nothing, FiberRef[Int]] = 
  FiberRef.make(
    initial = 0,
    fork = (parentValue: Int) => parentValue, // value for child fiber
    join = (parentValue: Int, childValue: Int) => math.max(parentValue, childValue)
  )
```

<Note>
  `FiberRef.make` returns a scoped resource that's automatically cleaned up when the scope closes. For unsafe creation, use `FiberRef.unsafe.make`.
</Note>

### Specialized FiberRefs

```scala
import zio._

// Set-based FiberRef with compositional updates
val setRef: ZIO[Scope, Nothing, FiberRef.WithPatch[Set[String], SetPatch[String]]] = 
  FiberRef.makeSet(Set.empty[String])

// Environment-based FiberRef
val envRef: ZIO[Scope, Nothing, FiberRef.WithPatch[ZEnvironment[Any], ZEnvironment.Patch[Any, Any]]] = 
  FiberRef.makeEnvironment(ZEnvironment.empty)

// Custom patch-based FiberRef
val patchRef: ZIO[Scope, Nothing, FiberRef.WithPatch[Value, Patch]] = 
  FiberRef.makePatch(initialValue, differ)
```

## Core Operations

### Reading and Writing

```scala
val fiberRef: FiberRef[Int] = ???

// Get current value
val current: UIO[Int] = fiberRef.get

// Set new value
val updated: UIO[Unit] = fiberRef.set(42)

// Get and set atomically
val previous: UIO[Int] = fiberRef.getAndSet(42)
```

### Atomic Updates

```scala
val fiberRef: FiberRef[Int] = ???

// Update with function
val increment: UIO[Unit] = fiberRef.update(_ + 1)

// Update and get new value
val newValue: UIO[Int] = fiberRef.updateAndGet(_ + 1)

// Get old value and update
val oldValue: UIO[Int] = fiberRef.getAndUpdate(_ + 1)
```

### Partial Updates

```scala
val fiberRef: FiberRef[Option[Int]] = ???

// Update only if defined
val updated: UIO[Unit] = fiberRef.updateSome {
  case Some(n) if n < 100 => Some(n + 1)
}

// Get and update only if defined  
val previous: UIO[Option[Int]] = fiberRef.getAndUpdateSome {
  case Some(n) => Some(n * 2)
}
```

### Modify with Return Value

```scala
val fiberRef: FiberRef[Int] = ???

// Atomically read and update, returning a value
val result: UIO[String] = fiberRef.modify { current =>
  (s"Value was $current", current + 1)
}

// Conditional modification
val conditional: UIO[Boolean] = fiberRef.modifySome(false) {
  case n if n < 100 => (true, n + 1)
}
```

## Scoped Changes

### Locally

Temporarily change a `FiberRef` value for a specific scope:

```scala
val fiberRef: FiberRef[String] = ???

val program = for {
  _      <- fiberRef.set("original")
  result <- fiberRef.locally("temporary") {
    for {
      value <- fiberRef.get // "temporary"
      _     <- someEffect
    } yield value
  }
  after <- fiberRef.get // "original" (restored)
} yield result
```

### Locally With Function

```scala
val fiberRef: FiberRef[Int] = ???

val program = fiberRef.locallyWith(_ + 10) {
  for {
    value <- fiberRef.get // original + 10
    _     <- someEffect  
  } yield value
}
```

### Scoped Changes

```scala
val fiberRef: FiberRef[String] = ???

val scoped = ZIO.scoped {
  for {
    _      <- fiberRef.locallyScoped("scoped value")
    result <- someEffect
  } yield result
  // Value automatically restored when scope closes
}
```

## Fork and Join Semantics

### Default Behavior

By default, child fibers inherit the parent's value, and on join, the child's value replaces the parent's:

```scala
for {
  ref   <- FiberRef.make(0)
  _     <- ref.set(1)        // parent = 1
  fiber <- ref.set(2).fork   // child inherits 1, sets to 2
  _     <- fiber.join        // parent becomes 2
  value <- ref.get           // value = 2
} yield value
```

### Custom Join Logic

You can customize how values are merged on join:

```scala
for {
  // Use max when joining
  ref   <- FiberRef.make(0, identity[Int], math.max)
  _     <- ref.update(_ + 2)  // parent = 2
  fiber <- ref.update(_ + 1).fork // child = 2 + 1 = 3
  _     <- ref.update(_ + 5)  // parent = 2 + 5 = 7
  _     <- fiber.join         // parent = max(7, 3) = 7
  value <- ref.get            // value = 7
} yield value
```

<Tip>
  The combining function should be associative for predictable behavior. If it's also commutative, joining multiple fibers concurrently will produce deterministic results.
</Tip>

## Built-in FiberRefs

ZIO provides several built-in `FiberRef` instances:

```scala
import zio._

// Current log level
val logLevel: FiberRef[LogLevel] = FiberRef.currentLogLevel

// Log annotations
val annotations: FiberRef[Map[String, String]] = FiberRef.currentLogAnnotations

// Log spans
val spans: FiberRef[List[LogSpan]] = FiberRef.currentLogSpan

// Metric tags
val tags: FiberRef.WithPatch[Set[MetricLabel], SetPatch[MetricLabel]] = 
  FiberRef.currentTags
```


```scala

```

## Integration with ThreadLocal

For integrating with Java code that uses `ThreadLocal`, you can convert a `FiberRef` to a `ThreadLocal`:

```scala
import zio._

val program = for {
  fiberRef   <- FiberRef.make("value")
  threadLocal <- fiberRef.asThreadLocal
  // Use threadLocal with Java APIs
  value = threadLocal.get()
} yield value
```

<Note>
  The `ThreadLocal` is backed by the `FiberRef` on ZIO-managed threads and behaves like an ordinary `ThreadLocal` on other threads. This requires enabling `Runtime.enableCurrentFiber`.
</Note>

## Advanced: Compositional Updates

For complex state that's built from pieces, use patch-based `FiberRef`:

```scala
import zio._

val program = for {
  // Set starts empty
  tags <- FiberRef.makeSet(Set.empty[String])
  
  // Fork child that adds tags
  fiber <- tags.update(_ + "child-tag").fork
  
  // Parent adds different tags
  _ <- tags.update(_ + "parent-tag")
  
  // Join merges both sets
  _ <- fiber.join
  
  // Result contains both tags
  result <- tags.get
} yield result
// result = Set("child-tag", "parent-tag")
```

## Best Practices

<Warning>
  The value inside a `FiberRef` should be immutable. While `Ref.Synchronized` supports mutable values with proper synchronization, regular `FiberRef` assumes immutable values.
</Warning>

<Tip>
  Use `FiberRef.locally` or `locallyScoped` instead of manual `getAndSet` + `set` pairs to ensure values are always restored, even on interruption.
</Tip>

<Card title="Performance Tip" icon="gauge-high">
  Regular `FiberRef` uses compare-and-swap operations for maximum performance. Only use `Ref.Synchronized` if you need to perform effects during updates.
</Card>
